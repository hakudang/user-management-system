# 要件定義書（System Requirements）

## 1. 目的
本書はシステム全体の要件を定義し、開発・テスト・運用の基準とする。

## 2. スコープ
対象となる機能範囲を明確化する。

## 3. 利用者一覧（User Types）
| 役割 | 説明 |
|------|------|
| Admin | 全権限 |
| Staff | 一部管理 |
| End User | 会員本人 |

## 4. 機能一覧（Functional Requirements）
| ID | 機能名 | 説明 |
|----|--------|------|
| FR-001 | 会員登録 | 基本情報入力〜確認〜完了 |
| FR-002 | ログイン | メール＋PWでログイン |

## 5. 非機能要件（Non-functional）
- 表示速度：LCP < 2.5s  
- 可用性：99%  
- セキュリティ：OWASP Top10準拠  

## 6. 前提/制約
- ブラウザ：Chrome / Safari / Edge  
- サーバ：Linux / PHP8 / MySQL8  

## 7. 用語定義
| 用語 | 意味 |
|------|------|
| 会員 | 登録ユーザー |
